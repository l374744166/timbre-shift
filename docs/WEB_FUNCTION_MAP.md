# Timbre Shift Web 功能地图

这份文档用于约束前端改动：先确认按钮真实用途，再调整展示。不要因为 UI 重排改变主功能含义。

## 工作台

### Seed-VC 快速试听

用途：用参考声音快速听音色方向，不训练模型。

主要按钮：

- `保存参考声音`
  - 接口：`POST /api/save-voice`
  - 用途：创建一个可被 Seed-VC 参考的音色，并保存上传素材。
  - 素材类型：
    - `干净人声`：直接保存。
    - `带伴奏，自动分离人声`：先 Demucs 分离，再保存分离后的人声。

- `补充参考素材`
  - 接口：`POST /api/add-voice-sample`
  - 用途：给已选音色补充参考声音素材。
  - 素材类型同上。

- `开始 Seed-VC 试听`
  - 接口：`POST /api/generate`
  - 关键字段：`engine_id=seedvc`
  - 要求：选择已有参考声音，或本次上传新声音；选择歌曲库歌曲，或本次上传新歌曲。

展示原则：

- 不显示 RVC 训练按钮。
- 不说“模型已训练”。
- 文案用“参考声音”“试听目标”“试听修饰”。

## Applio RVC 正式生成

用途：用已训练 RVC 模型正式换声，适合一个目标音色反复生成多首歌。

主要按钮：

- `创建 RVC 音色库`
  - 接口：`POST /api/create-voice-profile`
  - 用途：创建空音色库。
  - 注意：创建后没有素材，也没有模型，需要继续添加训练素材。

- `添加训练素材`
  - 接口：`POST /api/add-voice-sample`
  - 用途：给当前 RVC 音色库添加训练素材。
  - 当前工作台 RVC 入口默认按干净人声处理，避免和 Seed-VC 参考声音入口混淆。
  - 如果需要带伴奏分离，可在后续专门做 RVC 素材导入面板，不能混进 Seed-VC 控件。

- `开始 RVC 生成`
  - 接口：`POST /api/generate`
  - 关键字段：`engine_id=rvc_applio`
  - 要求：音色库已有 ready RVC 模型，或后端能找到该音色的 ready 模型。

展示原则：

- 必须显示素材数量。
- 必须显示已训练 RVC 模型数量。
- 训练过几个就显示几个，例如：`已训练 2 个 RVC 模型`。
- 文案用“音色库”“训练素材”“正式生成”“RVC 模型”。

## RVC 训练页

用途：单独管理 RVC 数据集准备和模型训练。

主要按钮：

- `准备数据集`
  - 接口：`POST /api/applio-prepare`
  - 用途：读取音色库中的 `clean_audio_path`，准备 Applio 数据集。
  - 要求：音色库至少有一个可用素材。

- `开始训练`
  - 接口：`POST /api/applio-train`
  - 用途：按选择轮数训练 RVC 模型。
  - 训练成功后写入 voice model 记录。

展示原则：

- 不展示完整模型技术路径作为主文案。
- 应显示“模型已保存”“可回工作台生成歌曲”。

## 音色库

用途：管理本地音色资源。音色库是 Seed-VC 和 RVC 共用的基础资源。

主要按钮：

- `新建音色`
  - 行为：跳到工作台，并切到 RVC 正式生成，用于创建 RVC 空音色库。

- `选择音色库`
  - 行为：选中该音色，并回到工作台。

- `训练模型`
  - 行为：选中该音色，切到 RVC 训练页。

展示原则：

- 卡片必须显示：
  - 音色名称
  - 素材数量
  - RVC 模型数量
  - 质量提示

## 歌曲库

用途：管理已保存歌曲和可复用分离结果。

主要按钮：

- `上传新歌曲`
  - 行为：跳回工作台歌曲上传区域。

- `选择`
  - 行为：选中歌曲并回到工作台。

展示原则：

- 显示歌曲是否已分离。
- 已有干净人声/伴奏缓存时，应该提示可跳过 Demucs。

## 结果区

主要按钮：

- `下载 MP3 / WAV`
  - 用途：下载带伴奏成品。

- `下载干声 MP3 / WAV`
  - 用途：下载转换后干声。

- `设为最终版本`
  - 接口：`POST /api/select-variant`
  - 用途：把对比版本复制为 final 输出。

- `标记喜欢`
  - 接口：`POST /api/variant-feedback`
  - 用途：记录用户偏好的版本。

- `保存为该音色默认参数`
  - 接口：`POST /api/voice-preference`
  - 用途：保存当前音色的默认生成参数。

## 右侧栏

主要按钮：

- `停止任务`
  - 接口：`POST /api/cancel-task`

- `演示推荐配置`
  - 行为：回到工作台。

- `进入 RVC 训练`
  - 行为：切到 RVC 训练页。

## 环境检查

主要按钮：

- `重新检测`
  - 接口：`GET /api/check`
  - 检查项：ffmpeg、Demucs、Seed-VC、Python 环境。

## 共享素材逻辑

- Seed-VC 和 RVC 共用同一套音色库素材。
- `POST /api/save-voice` 和 `POST /api/add-voice-sample` 最终都会写入 voice samples。
- RVC 数据集读取的是 voice sample 的 `clean_audio_path`。
- 因此：Seed-VC 保存的人声素材、带伴奏分离后的人声素材，都可以被 RVC 训练使用。
- RVC 添加的 clean 素材，也可以被 Seed-VC 用作参考声音。

## 改动红线

- 不要把 Seed-VC 和 RVC 的按钮混在一起。
- 不要让 RVC 主流程出现 Seed-VC 的“参考声音”文案。
- 不要让 Seed-VC 主流程出现 RVC 的“训练模型”文案。
- 不要隐藏 RVC 已训练模型数量。
- 不要显示无动作按钮；按钮出现就必须有绑定行为。
